## API月度汇总修复

### 问题
`queryMonthBillList()` 函数使用 `settleDate=月初-01&dateType=2` 只查询每月1日数据，导致返回金额与Excel文档差40~46倍。

### 解决方案
改用 API 文档支持的 `beginDate/endDate` 区间查询：
- beginDate = `YYYY-MM-01 06:00:00`（月初）
- endDate = `YYYY-MM+1-01 06:00:00`（次月月初，覆盖完整营业月）

### 修改文件
- `/Users/zhangrunjie/WorkBuddy/Claw/index.html` 第610-647行

### 关键代码变更
```js

// 新（正确）
const beginDate = `${monthStr}-01 06:00:00`;
const endDate = `${nextYear}-${p(nextMonth)}-01 06:00:00`;
const qs = `...beginDate=${encodeURIComponent(beginDate)}&endDate=${encodeURIComponent(endDate)}`;
```

## 数据准确性修复（第三轮 - v21）

### 问题
用户反馈：
1. 数据仍然不准确 - 广安门店2025年显示 **8,179,666.81**，正确值应为 **9,644,444.31**
2. Console中出现大量 `24天失败`、`23天失败` 等异常日志

### 根因分析

#### 问题1：缓存清理后旧数据仍被恢复
虽然 `lsClearOldCache()` 清空了 localStorage，但执行顺序有问题：
1. `loadAllData()` 先调用 `lsClearOldCache()` 清理缓存
2. 然后调用 `lsRestoreMonths()` 从 localStorage 恢复数据
3. 但 `loadedMonths` 在清理时已被清空，而 `lsRestoreMonths()` 没有检查缓存版本
4. 如果 localStorage 清理不彻底（某些浏览器异步写入），仍会恢复旧数据

**真正根因**：`lsRestoreMonths()` 没有检查缓存版本是否匹配，导致在版本升级后仍可能恢复旧缓存。

#### 问题2：API并发过高导致大量请求失败
`queryMonthBillList()` 内部使用 `batchQuery(dayTasks, 3)` 逐日查询一个月的数据：
- 28家门店 × 31天 = 868个请求
- 并发为3时，同时有3个请求在飞
- 但预加载阶段 `batchQuery(tasks, 8)` 并发为8，多个门店同时查询月度数据
- 叠加后并发请求过多，触发API限流，导致大量请求超时失败

### 修复方案

1. **v21 升级缓存版本** - 强制重新清理
2. **修复 lsRestoreMonths()** - 添加缓存版本检查，版本不匹配时不恢复任何缓存
3. **修复 lsClearOldCache()** - 确保 `loadedMonths.clear()` 在设置版本号之前执行
4. **降低并发限制** - 所有 `batchQuery` 调用月度查询的并发从 8/10 降低到 3，逐日查询从 3 降低到 2

  - 第947-983行：修复 `lsClearOldCache()` 执行顺序
  - 第849-894行：修复 `lsRestoreMonths()` 添加版本检查
  - 第634行：逐日查询并发从 3 降到 2
  - 第1214行：预加载并发从 8 降到 3

## 首次加载体验优化（v22 - 修复）

### 问题
清除缓存后首次加载，主内容区域仍然提前显示，loading mask 没有正常工作。

### 根因分析
`loadAllDataWithCache({ progressive: false })` 虽然设置了非渐进式模式，但函数内部的预加载 `Promise.all(preloadPromises)` 没有被 `await`，导致函数在预加载完成前就返回了。然后 `.then()` 回调隐藏 loading mask、显示主内容，但后台数据还在加载中。

### 修复方案
将 `Promise.all(preloadPromises)` 赋值给变量 `preloadAll`，在非渐进式模式下 `await preloadAll`，确保所有数据加载完成后再返回。

### 修改文件
- `/Users/zhangrunjie/WorkBuddy/Claw/index.html`
  - 第1281-1292行：`Promise.all(preloadPromises)` 改为 `const preloadAll = Promise.all(...)`，并添加 `if (!progressive) await preloadAll;`

## 出租率报表自动加载修复

### 问题
用户反馈切换到"出租率"维度时，需要手动点击"查询"按钮才能显示数据，希望默认自动加载当前日期的数据。

### 根因分析
`initOccupyState()` 函数使用了 `availableYears` 变量，但该变量是 `initSelectors()` 函数内的局部变量，导致 `initOccupyState()` 抛出 `ReferenceError: availableYears is not defined` 异常，进而导致自动查询失败。

### 修复方案
1. **修复 `availableYears` 引用** - 改为使用 `DATA?.availableYears || [occNow.getFullYear()]`
2. **默认设置日维度** - 添加 `STATE.occDim = 'day'` 和 `selOccupyDim.value = 'day'`
3. **确保自动查询** - 切换到出租率tab时自动调用 `renderOccupancy()`
