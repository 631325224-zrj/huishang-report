## Claw 报表项目 - v25 修复提交（commit 5143938）

### 用户反馈
- 截图1：所有数据全0，显示"正在后台加载数据..."
- 截图2：2026年=0，2025年=3639万（只有11-12月数据），月度走势只有11-12月有数据
- 用户要求："自行核对完数据是否正确再给我看，加载方式也还不对"

### 根因分析

**Bug 1（致命）：`queryMonthBillList` catch块中把错误结果写入了 `SHOP_DATA_CACHE`**
```javascript
} catch (e) {
    apiStatus = `error: ${e.message}`;
    // 没有return！继续执行到函数末尾
}
// 这里把 income=0 的结果缓存了，后续不再重试，数据永久为0
SHOP_DATA_CACHE[cacheKey] = result;
```
当API调用失败（网络错误、超时、token过期）时：
1. catch块没有return，继续执行到函数末尾
2. 把 `income=0` 的结果写入 `SHOP_DATA_CACHE`
3. 后续对该门店该月份的请求直接从缓存返回0
4. 数据永久为0，永不重试

**Bug 2（加载方式）：第三阶段预加载308次请求（28店×11月）**
- 大量并发请求容易因网络/token问题批量失败
- 失败后由于Bug 1，所有失败门店数据永久为0
- 用户看到"正在后台加载数据..."但加载完成后数据还是0

### 修复内容

1. **`queryMonthBillList` 核心修复**
   - API错误码≠0时：返回 `{success: false}`，不写入缓存
   - catch网络错误时：返回 `{success: false}`，不写入缓存
   - 成功时：返回 `{success: true}`，写入缓存

2. **移除第三阶段预加载**
   - 不再初始加载所有历史月份
   - 改为纯按需加载：用户切换月份/年报时自动调用 `ensureMonthData`

3. **`ensureMonthData` 修复**
   - 识别 `success: false` 的结果，计入失败数
   - 只有全部门店成功（failCount===0）才标记 `loadedMonths.add(key)`
   - 有失败时下次自动重试

4. **`loadAllData` 第二阶段修复**
   - 识别 `success: false`，统计失败门店数
   - 有失败时不标记为已完整加载，刷新后自动重试

5. **`availableMonths` 修复**
   - 有缓存时不被 `loadedMonths` 限制
   - 用户可选择任意月份，未加载的自动触发 `ensureMonthData`

6. **缓存版本 v24→v25**
   - 强制清除旧脏缓存

### 已推送
- commit 5143938 已推送到 GitHub Pages
- 首次访问会自动清除旧缓存（v24→v25），重新加载当月数据

## Claw 报表项目 - v26 修复提交（commit 01ab3b1）

### 用户明确需求
用户提供需求文档，明确要求：
1. **首次加载**：必须加载本年和上一年的全部数据，可以接受加载慢，进入后要可以直接呈现准确数据
2. **后续缓存加载**：凭借缓存数据可以快速进入，但要避免出现新旧数据累加问题

### v25 的错误
v25 把第三阶段预加载全删了，改为"纯按需加载"，矫枉过正。用户打开页面后只能看到当月数据，年报和历史月份全部为空，必须手动切换才触发加载。

### v26 修复内容

1. **恢复第三阶段预加载（本年+去年所有月份）**
   - 生成近24个月范围（去年1月→今年当前月）
   - 利用 `loadedMonths.has(k)` 跳过已缓存月份
   - 最多加载24个月（远少于v25的308次）

2. **分小批量加载**
   - 每批6个月（6月×28店=168次并发），分2-4批
   - 每批之间 await，分批加载避免网络超时

3. **单月重试机制**
   - 某月首次加载失败后，等待2秒重试1次
   - 2次都失败才跳过，避免单次网络抖动导致永久失败

4. **保留 v25 Bug1 修复**
   - `queryMonthBillList` catch块不缓存失败结果，保留

5. **缓存版本 v25→v26**
   - 强制清除旧缓存，v26用户首次加载时重新拉取所有月份

6. **新、旧数据累加问题**
   - 预加载前：当前月数据已通过 Phase2 清零+重新填入
   - `ensureMonthData` 内：每个月份开始时清零该月所有数据（`DATA.monthlyData[key] = 0`）再重新累加
   - 两层清零机制确保不重复累加

## Claw 报表项目 - v26 fix（commit ea96630）

### 用户反馈
- 数据准确了（2026年=1298万），但 2025年=0
- 月度走势图只有 2026年5月有数据，其他为0
- 底部显示"正在后台加载数据，加载完成后自动更新..."

### 根因
v26 第三阶段写成 `(async () => { ... })()` —— **异步 IIFE 不等待**。`loadAllData()` 不等历史月份加载完就 return，loading-mask 提前消失。页面只显示当月数据，2025年历史数据还在后台慢慢加载。

### 修复
在 `(async () => { ... })()` 前面加 `await`：
```javascript
await (async () => {
  // 第三阶段预加载历史月份...
})();
```
现在 `loadAllData()` 会等待全部历史月份加载完成后再 return，loading-mask 直到数据全部就绪才消失。
