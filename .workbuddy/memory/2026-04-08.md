# 2026-04-08 工作日志

## 修复：localStorage脏缓存 + 预加载上月数据

### 问题1：脏缓存 key 错误
- **现象**：Console 显示 `[月数据] {2026-04: 774317, 2025-03: 757017}`，`2025-03` 的 key 错误，应为 `2026-03`
- **原因**：历史 bug 把 `2026-03` 以错误 key 写入了 localStorage
- **修复**：`lsRestoreMonths` 增加 key 合法性校验，自动清除年份超范围/格式错误/未来月份的脏 key

### 问题2：2月份数据加载慢
- **原因**：初始加载只覆盖当前月（4月），切到环比2月才触发按需加载
- **修复**：在 `loadAllData` 第三阶段预加载上个月（`prevMonthKey`），页面刷新时 3月和2月都加载完毕

### 文件变更
- 修改：`lsRestoreMonths` 增加脏 key 自动清理
- 修改：`loadAllData` 增加第三阶段预加载上月逻辑

## 修复（第二轮）：同比请求无效年份导致数据错乱

### 根本原因
- 初始 `STATE.compare = 'yoy'`，`STATE.yearB = 2025`（去年同期）
- `render()` 触发 `ensureMonthData(2025, monthB)`，发出 `settleDate=2025-02-01` 的 API 请求
- **API 不支持真实历史年份**，返回的实际上是当前月（2026-02）的数据，但被存入了 `2025-02` 这个错误 key
- 导致 `DATA.monthlyData['2025-02']` 有值，而 `DATA.monthlyData['2026-02']` 可能被重复叠加

### 修复
1. `ensureMonthData` 头部增加年份有效性校验：`year >= API_CONFIG.onlineYear && year <= nowY && 非未来月`，不满足直接跳过
2. `render()` 里同比（yoy）模式不再触发 `ensureMonthData`（API 不支持，只显示 —），只有环比（mom）才触发
