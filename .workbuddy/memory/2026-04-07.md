# 2026-04-07 工作日志

## 环比月份年份计算错误修复

### 问题
- 现象：2026年1月 环比显示的是 2026年12月（不存在），导致数据为 0
- 根本原因：`updateBSelector` 函数里，环比时 `STATE.yearB = STATE.yearA`，年份直接等于当期年份，没有考虑1月→上一年12月的跨年情况

### 修复
```js
// 修复前
STATE.yearB = STATE.yearA;
STATE.monthB = STATE.monthA === 1 ? 12 : STATE.monthA - 1;

// 修复后
STATE.monthB = STATE.monthA === 1 ? 12 : STATE.monthA - 1;
STATE.yearB = STATE.monthA === 1 ? STATE.yearA - 1 : STATE.yearA;
```

### 附加修复
- `selCompare` change 事件增加 `ensureMonthData(STATE.yearB, STATE.monthB)` 调用，切换到环比后立即触发对比月数据加载

## 环比历史月份数据始终为 0 的根本原因与修复（第二轮）

### 问题分析
- 现象：3月、2月、1月的环比数据全部显示 0.00，console 显示 `[2026-03] 缓存命中:0 需请求:840`
- 根本原因：`ensureMonthData` 原来逐天逐门店发 API 请求：`28天 × 28门店 = 784 次`，API 大量超时
- 之前的"缓存优化"方案（优先读 SHOP_DATA_CACHE）实际上无法命中缓存，因为初始 35 天窗口在时区或格式方面存在差异，或者超出 35 天范围的月份（如 1 月）根本没有缓存

### 终极修复
将 `ensureMonthData` 从**日维度 API**（784次请求）改为**月维度 API**（28次请求）：
- 每个门店只需查一次月汇总：`queryRevenue(settleDate, 'month', centerId, shopId)`
- `settleDate` 传当月1日（如 `2026-02-01`），API 返回整月数据
- 请求量减少 96%（784→28），彻底解决超时问题

### 代码位置
`ensureMonthData` 函数（约第 595 行）

## 静态数据方案废弃（11:00）

### 决策
用户明确：**不能把 Excel 作为固定历史数据**，原因：
- 财务可能调整历史账单，每次调整都要重新导出导入，极麻烦
- 应该保持纯 API 拉取，数据始终与收银系统同步

### Excel 数据用途
- 仅用于**验证 API 准确性**（日维度汇总 vs 月维度一致 ✅）
- 确认字段对应：`营业收入 - 非收入金额 = 纯收金额`
- 确认门店代码对应：Excel 1-28 = centerId 247413-247440

## 纯 API + localStorage 缓存优化（最终方案，11:10）

### 核心策略
| 优化 | 效果 |
|------|------|
| 月维度 API 替代日维度 | 784次 → 28次/月（96%减少） |
| localStorage 持久化 | 第二次访问历史月份 0 请求 |
| 初始加载精简 | 35天 → 7天日数据 |
| 当月用月维度 API | 精确汇总，无日维度累加误差 |

### 初始加载请求量
- 近 7 天日数据：7 × 28 = **196 次**
- 当月月维度：28 次
- 合计：**224 次**（vs 优化前 980 次，减少 77%）

### localStorage 缓存结构
- `hs_monthly_cache`: `{ 'YYYY-MM': { total, shops: { cid: amount }, ts } }`
- `hs_monthly_meta`: `{ loadedMonths: [...] }`
- 首次访问：加载历史月份时自动写入 localStorage
- 再次访问：从 localStorage 恢复，无需任何 API 请求

### 文件变更
- 删除：`static-data.js`（废弃）
- 修改：`loadAllData` 重写，新增 localStorage 工具函数
- 修改：`ensureMonthData` 增加 localStorage 写入
- 修改：`initSelectors` 月份选择器只显示有数据的月份（disabled 无数据的月份）
- 修改：新增 `updateMonthOpts` 动态刷新月份选项
- 修改：`updateBSelector` 自动跳过无数据的月份

### 月份过滤逻辑
- 2025年：1-5月 disabled，6-12月可选
- 2026年：1-3月 disabled，4月可选
- 同比切换：若目标月份无数据，自动降级到最近有数据的月份
- 环比切换：若上月无数据，自动降级到前一年12月，再逐月往前找
