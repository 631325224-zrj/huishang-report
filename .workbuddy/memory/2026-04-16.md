## getServiceArea 接口参数问题修复

**问题**：`queryShopRooms(centerId)` 调用 `getServiceArea` 时只传了门店 centerId，导致返回 -400003（token 无效）。

**根本原因**：接口文档确认 `centerId` 参数含义是"餐饮集团ID"（247412），不是门店 ID。`shopId` 才是门店 centerId（可选参数）。

**修复**：
- `queryShopRooms` 参数从 `centerId` 改为 `shopId`，内部固定用 `API_CONFIG.centerId`（247412）作集团 ID
- Query String 改为 `centerId=247412&shopId=${shopId}&pageNo=1&pageSize=200`
- 调试函数也同步加上 shopId=247413（朝阳店）

**状态**：已推送。但测试仍报 -400003，原因未明。
