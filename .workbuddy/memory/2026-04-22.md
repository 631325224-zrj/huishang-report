# 2026-04-22

## 年报数据修复（月度查询方案最终确定）

**问题**：年报表头显示日期（2026-04-22）而非年份，数据全为 0。

**根本原因**：
1. 表头正确，只是年报正在异步加载中（`ensureYearData` 返回 true 后 render 提前 return），页面遗留了上次日报的表头。
2. 月度查询改用了逐日累加（每门店31次请求），导致年报加载极慢。

**解决方案**：
- 月度查询回归 `settleDate + dateType=2`（月维度），1次请求/门店，直接返回整月 `income.incomeTotal`
- 删除 `queryOneDayIncome` 和逐日累加逻辑（90行代码变5行）
- 清理 `render()` 和 `fetchShopData()` 中残留的调试面板代码

**2026-04-24 补充**：月度查询参数错误根因找到——`dateType=2` 是周维度，`dateType=3` 才是月维度！已修正。
