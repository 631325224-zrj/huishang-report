## 年报渲染逻辑重构

### 问题1：表格消失
- **根本原因**：`renderSummary` 年报分支 `return` 后，`render()` 里 `renderTable(shops)` 仍然执行了第一次（可能是空数据），然后异步完成后回调 `renderTable()` **无参数**，`shops` 为 undefined，导致报错清空表格
- **修复**：重构为 `ensureYearData(onReady)` 函数，统一判断是否需要异步等待；`render()` 里年报维度调用 `ensureYearData(() => render())`，若还在加载则 `return`，等回调后完整重新 render

### 问题2：2025年8月折线图=0
- **原因**：8月不在预加载范围（只预加载近6个月），切换年报时才触发加载，加载期间显示0是正常过渡状态
- **若持续为0**：localStorage 可能有脏缓存，执行 `localStorage.clear()` 后刷新即可

### 代码位置
- `ensureYearData()` 函数：`renderSummary` 前新增
- `render()` 函数：1624行附近，`renderSummary()` 后增加年报判断分支

## 年报切换卡死 bug 修复（2026-04-14 第二次）

### 两个根本原因
1. **每月完成都触发 render**：`ensureMonthData` 完成时调 `render()`，11个月同时加载就触发11次重绘，卡死UI线程。修复：改为 `if (loadingMonths.size === 0) render()`，批量加载完成后只渲一次。
2. **防并发逻辑返回 undefined 导致 Promise.all 提前 resolve**：`loadingMonths.has(key)` 时直接 `return`（返回 undefined），`ensureYearData` 里 `Promise.all([undefined, ...])` 立即 resolve，触发 `onReady()` 此时数据未就绪，`ensureYearData` 再次被调用形成循环。修复：改为返回轮询 Promise，等 `loadedMonths` 标记后才 resolve。

### 教训
- `Promise.all` 里如果某个 Promise 提前 resolve（如返回 undefined），整个批次会"假完成"
- 批量异步加载时，不要每次完成都触发 render，要等全部完成后只渲染一次

## 年报表头显示日报格式（2026-04-14 第三次修复）

- `render()` 里 `renderSummary()` 在 `ensureYearData()` 检查之前执行

## 根因分析 & 修复（2026-04-14）

### 问题
xlsx收银系统数据与CSV报表数据差异，阜阳/昆山/滁州/蚌埠/罍街等门店特定月份少计几千~几万元。

### 根本原因
旧版 `queryMonthBillList` 使用 `getBusinessSituation` 日汇总 API，该 API 返回的数据比收银系统直接导出少（可能原因：日汇总 API 在某些场景下数据不完整）。
收银系统 xlsx 使用 `sum(last_total) - sum(not_income_money)` 直接从账单明细统计，才是正确口径。

### 修复方案
1. **改用 `getserialdata` 明细 API + 分页查询**：每次查询一天 + 全量分页取账单明细，计算 `sum(last_total) - sum(not_income_money)`，与收银系统口径完全一致
2. **升级缓存版本 v4→v5**：强制清除旧缓存，强制用新 API 重建所有月份数据
3. 新代码加了日缓存（`serial_${date}` key）避免同一天重复分页查询

### 修复后操作
刷新页面后会自动清除旧缓存并用新 API 重新加载所有月份数据。加载完成后数据应与收银系统 xlsx 完全一致。
