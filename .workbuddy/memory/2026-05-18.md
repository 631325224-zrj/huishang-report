## 出租率首次加载优化（全局loading期间预加载 + Promise等待）

**问题**：首次打开页面，全局 loading 等的是营收数据，不包含出租率，所以切换到出租率Tab还要再等一次查询。

**解决方案**：
1. 在 `loadAllDataWithCache`（无缓存路径 `progressive: false`）的 `await preloadAll` 之后，预加载今日出租率
2. 在有缓存秒进路径中，后台并行预加载今日出租率
3. 引入全局 `_OCC_PROMISE` 变量缓存正在进行的预加载 Promise
4. `renderOccupancy()` 三种状态处理：
   - `useCache`：缓存已就绪 → 直接渲染
   - `waitPromise`：预加载进行中 → `await _OCC_PROMISE` 等待完成，不显示"查询中…"
   - 否则：正常发请求，显示轻量加载提示
5. `DATA.occToday` 提前在 Promise 外部设置，确保 `waitPromise` 日期匹配条件正确

**修改位置**：
- 全局变量：新增 `_OCC_PROMISE`
- `loadAllDataWithCache`（无缓存路径）：`DATA.occToday = today` 提前设置 + `_OCC_PROMISE` 缓存
- 有缓存秒进路径：`DATA.occToday` 提前设置 + `_OCC_PROMISE = getToken().then(...)` 立即创建
- `renderOccupancy()`：三种状态分支逻辑

## 四合一修复（2026-05-18 17:35）

### 1. 数据查询方式修复
**问题**：2025年数据仅3,351,096.39（全部门店），明显偏低（广安单店7月就应1,200,000+）
**根因**：`dateType=2 + settleDate=YYYY-MM-01` 实际返回的是单日数据，不是整月汇总
**修复**：恢复 `2fea456` 的正确查询方式：
- **历史年（year < curYear）**：`beginDate=YYYY-MM-01&endDate=YYYY-MM-31&dateType=1`（区间查询整月）
- **当年（year == curYear）**：`settleDate=YYYY-MM-01&dateType=3`（月维度汇总）

### 2. 加载机制修复（缓存优先）
**问题**：每次打开都等全局加载完才呈现，无缓存秒进体验
**修复**：
- 有缓存时：立即从 localStorage 恢复月份数据 → 累加年度数据 → 立即 render() 呈现 → 后台静默调用 loadAllData() 刷新最新数据
- 无缓存时：保持 loading-mask，等 loadAllData() 完成再一次性呈现

### 3. 出租率自动加载
**问题**：切换到出租率 Tab 后数据为空，需要手动点"查询"
**修复**：Tab 切换逻辑中，当 `STATE.dim === 'occupy'` 且 `OCC_DATA` 为 null 时，自动调用 `renderOccupancy()`

## 2025年历史数据为0 + 加载分步渲染修复（2026-05-18 17:00）

**问题1**：清除缓存后2025年数据始终为0
- **根因**：`queryMonthBillList` 中对历史年份（year < curYear）使用了 `beginDate/endDate + dateType=1`，该 API 不支持区间查询，返回收入为0
- **修复**：统一使用 `settleDate=YYYY-MM-01&dateType=2`，历史年和当年同一套查询逻辑
- **缓存版本**：v20 → v21，强制清除旧脏缓存

**问题2**：数据分步渲染，进入后逐步出现
- **根因**：`loadAllData` 各阶段和预加载每完成一个月都调用 `render()`，页面先显示空框架再逐步填充
- **修复**：
  1. 初始化不立即隐藏 `loading-mask`
  2. `loadAllData` 内部去掉所有中间 `render()` 调用
  3. 第三阶段 `Promise.all(preloadPromises)` 改为 `await`，等全部完成
  4. 全部完成后一次性隐藏 mask 并 `render()`
  5. 年报进入时直接呈现完整数据，不再有"正在后台加载数据"提示
