# 2026-05-07 工作记录

## Bug修复：年报数据偏大158亿 + 加载卡死

### 问题1：预加载双重累加（2025年=158,266,529 明显偏大）
- **根因**：第三阶段预加载循环中同时写了本地变量 `monthlyData` 和 `DATA.monthlyData`，而 `DATA.monthlyData = monthlyData` 是同一引用，导致每次写入相当于两倍
- **修复**：删掉预加载里重复写 `DATA.*` 的代码，只写本地变量即可

### 问题2：年报加载卡死（2025年12月×28店×逐天 = 1万次请求）
- **根因**：v10改回 `getserialdata` 逐天查历史年，每月31天×28店=868次，12个月=10416次，全部超时
- **真相**：上次"getBusinessSituation历史年返回0"的结论是错误的！探测截图显示 2025年 dateType=2+settleDate=2025-09-01 返回 **54,394.95**（整月汇总），完全有效
- **修复**：历史年和当年统一用 `getBusinessSituation dateType=2 + settleDate=YYYY-MM-01`，28店每月只需28次请求
