# 2026-05-12 工作记录

## huishang-report 年数据Bug修复（v11）

### Bug1：预加载双重累加（2025年数据约1.58亿，正常应约1000万）
- 原因：第三阶段预加载同时写本地变量和 `DATA.*`，但两者是同一引用，导致数据翻倍
- 修复：预加载只写本地变量（`monthlyData/yearlyData/shopMonthlyData/shopYearlyData`），不重复写 `DATA.*`

### Bug2：年报加载卡死（门店表格一直显示"正在后台加载数据"）
- 原因1：v10版本错误地将历史年改为 `getserialdata` 逐天查询（12月×28店=10000+次请求，全部超时）
  → 修复：历史年统一恢复用 `getBusinessSituation dateType=2 + settleDate=YYYY-MM-01`
- 原因2：`let loadedMonths` 声明在第1225行，但在第917行就被 `lsClearOldCache()` 访问，触发 TDZ ReferenceError
  → 修复：将 `let loadedMonths` 声明移到第789行（在所有使用它的代码之前）
- 原因3：`ensureYearData` 的 `Promise.all().then()` **没有 `.catch()`**，若 Promise 拒绝则 `.then()` 永不执行，`_yearBatchLoading` 永远卡死
  → 修复：添加 `.catch()` + 安全超时60秒强制渲染
